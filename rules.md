# Project Rules

このファイルは、このリポジトリでの作業ルールを記録するためのもの。
新しい指示が出たら追記・更新する。

## Content / Direction

- このサイトは「GAIさんの記録場所」として扱う。
- 基本用途は、アイデア、作ったもの、メモ、リンクの整理。
- デザインはシンプル、ミニマル、ダークテーマ寄りを維持する。
- 強いランディングページ感や大きすぎるヒーロー見出しは避ける。
- 長めのアイデアや重要な項目は、一覧だけでなく詳細ページでも読める構造にする。
- 外部リンクは、表示するだけでなくクリックして飛べるようにする。

## Content Updates

- ユーザーが送ったアイデアは `Ideas` に追加する。
- ユーザーが送った作ったもの / リポジトリは `Projects` に追加する。
- 必要に応じて、ステータス（例: `制作中`）を表示できるようにする。
- Gumroad など販売ページがある場合は、関連プロジェクトに紐づけて記録する。
- 追加した内容は、一覧カードと詳細ページの両方で読める形に整える。

## Workflow

- このプロジェクトでは、通常の変更は確認せずにすぐ `commit` / `push` してよい。
- 更新後にキャッシュ問題が起きやすいので、必要に応じてアセットのバージョンを更新する。
- 変更は GitHub Pages 公開前提で壊れない静的構成を維持する。
